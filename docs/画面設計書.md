# 画面設計書

紅白予想アプリのフロントエンド（React + Vite の SPA）。実装の正は [`web/src/`](../web/src/)。
画面数が少ないため、ルータを使わず**タブ切替の単一ページ**で構成する。

## 1. 画面構成

```
┌─────────────────────────────────────────────┐
│ AuthBar  紅白予想              [ユーザー名/ログイン] │  ← 常時表示
├─────────────────────────────────────────────┤
│  [ 予想 ] [ ランキング ]                       │  ← タブ（nav.tabs）
├─────────────────────────────────────────────┤
│                                             │
│   選択中タブのパネル（main）                    │
│                                             │
└─────────────────────────────────────────────┘
```

- 画面ルート: [`App.tsx`](../web/src/App.tsx)。`tab` 状態（`predict` / `ranking`）を
  `useState` で保持し、対応するパネルを描画する。
- サーバーデータは全て TanStack Query で取得・キャッシュする（`web/src/lib/api.ts` 経由）。
- スマホ幅でもレイアウトが大きく崩れないこと（レスポンシブ）。

## 2. 共通コンポーネント

| コンポーネント | 役割                                                                                                 |
| -------------- | ---------------------------------------------------------------------------------------------------- |
| AuthBar        | ヘッダ。ログイン状態に応じて「Googleでログイン」/ ユーザー名・アバター・ログアウト・表示名編集を出す |
| ConfirmDialog  | ネイティブ `confirm` を置き換える確認ダイアログ（予想の取消などで使用）                              |
| useMe          | `GET /api/auth/me` を購読するフック。未ログインは `null`                                             |

## 3. 画面詳細

### 3.1 ヘッダ（AuthBar）

| 状態         | 表示                                                          |
| ------------ | ------------------------------------------------------------- |
| 未ログイン   | 「Googleでログイン」リンク（`/api/auth/google/login` へ遷移） |
| ログイン済み | アバター・表示名、表示名編集、「ログアウト」ボタン            |

- ログアウト: `POST /api/auth/logout` → `me` を無効化し未ログイン表示へ。
- 表示名編集: `PATCH /api/auth/me`。1〜50文字。

### 3.2 予想タブ（PredictionsPanel）

ログイン要否で表示を出し分ける。

| 状態                      | 表示                                                   |
| ------------------------- | ------------------------------------------------------ |
| 未ログイン                | 「ログインしてください」導線                           |
| 受付中（`season.isOpen`） | 予想フォーム＋予想一覧                                 |
| 締切後                    | フォームは出さず、予想一覧のみ（または締切メッセージ） |
| シーズン無し              | その旨を表示                                           |

**予想フォームの操作フロー**:

1. **アーティスト検索**（ArtistSearch）: 入力に応じ `GET /api/artists/external?q=` を呼び、
   候補（画像＋名前）を表示。クリックで選択。
2. **曲検索/選択**（SongSearch）: 選択アーティストに対し `GET /api/songs/external?q=` で曲を検索。
   「（曲は予想しない）」を選べば出場予想のみになる。
3. **確信度**: 1〜5 を選択。
4. **コメント**: 任意（最大500文字）。
5. 「予想する」→ `POST /api/predictions`。成功で一覧へ追加。

**エラー表示**（`ApiError.code` で分岐し日本語メッセージを表示）:

| code             | 表示意図                           |
| ---------------- | ---------------------------------- |
| CONFLICT         | 「このアーティストは既に予想済み」 |
| SEASON_CLOSED    | 「受付は終了しています」           |
| VALIDATION_ERROR | 入力内容を見直す旨                 |

**予想一覧**（`GET /api/predictions?seasonId=`）:

- 各行に アーティスト名 / 曲名 / ユーザー名 / 確信度 / コメント を表示。
- 自分の予想のみ「取消」ボタン（ConfirmDialog 経由で `DELETE /api/predictions/:id`）と編集が可能。
- 他人の予想には操作ボタンを出さない（API側でも 403 FORBIDDEN）。

### 3.3 ランキングタブ（RankingsPanel）

「今シーズン / 通算」のサブタブで切り替える（`.subtabs`、`useState` で `scope` を保持）。

- **今シーズン**: `GET /api/seasons/current` → そのシーズンの `GET /api/rankings/:seasonId`。
- **通算**: `GET /api/rankings/overall`（締切済み全シーズンの合算）。

| 状態                 | 表示                                       |
| -------------------- | ------------------------------------------ |
| シーズン無し（今シーズン） | 「シーズンがありません。」            |
| ローディング         | 「集計中…」                                |
| 締切前（`CONFLICT`、今シーズン） | 「結果確定後に表示されます（締切前）。」 |
| 空（通算で締切済みなし等） | 「まだランキングはありません。」       |
| 表示                 | 順位 / ユーザー / スコア / 的中 のテーブル |

- 並び順はサーバー側で確定（スコア降順 → 的中件数 → 投稿の早さ）。同点は同順位。
- 通算スコアは負もあり得る（そのまま表示）。

## 4. 状態遷移（シーズンのライフサイクルと画面）

```
受付中(close_at=NULL/未来)        締切後(close_at<=now)         結果確定後
─────────────────────────▶─────────────────────────▶─────────────────────────▶
予想: 投稿/編集/取消 可            予想: 一覧のみ(投稿403)        予想: 一覧のみ
ランキング(今季): 締切前メッセージ   ランキング(今季): 集計表示      ランキング(今季): 集計表示
ランキング(通算): 過去分のみ合算     ランキング(通算): 当季も合算    ランキング(通算): 当季も合算
```

## 5. API クライアント方針

- `web/src/lib/api.ts` が `apiGet` / `apiPost` / `apiSend(PUT|PATCH|DELETE)` を提供。
  すべて `credentials: include`（セッションCookie送信）。
- エラーは `ApiError(status, code, message)` に正規化し、画面側は `code` で分岐して日本語表示する。
- サーバーデータは Zustand 等へコピーせず TanStack Query のキャッシュをそのまま使う。
  UIローカル状態（タブ・選択中候補・入力値）は `useState`。
