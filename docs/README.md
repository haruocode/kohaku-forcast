# 設計書一覧

NHK紅白歌合戦の出場者・歌唱曲・結果を予想するWebアプリ「紅白予想（kohaku-yoso）」の設計書一式。

プロダクトの全体方針・思想は [`/AGENTS.md`](../AGENTS.md) を正とする。本ディレクトリの各設計書は、
AGENTS.md の方針を前提に、現行コードの実装内容を設計レベルで記述したもの。

## 用語

- **シーズン (season)**: 紅白の年度。受付開始 → 締切（公式発表） → 結果確定 のライフサイクルを持つ。
- **受付中 / 締切**: `prediction_close_at` が `NULL` か未来なら受付中。締切は運営が「公式発表の日時」を後から設定する。
- **遅延アップサート (find-or-create)**: ユーザーが外部音楽DB(Spotify/MusicBrainz)から選んだ
  アーティスト/曲を、`(source, external_id)` をキーにローカルDBへ初回参照時に取り込む方式。
- **早押しボーナス**: 早く予想したほど基礎点に倍率が掛かる仕組み（最大1.5倍）。
- **サイト内ポイント**: 予想の採点スコア。シーズンポイントと通算ポイントを持つ。譲渡・換金・購入いずれも不可で、財産的価値を持たせない。
